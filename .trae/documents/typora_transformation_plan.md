# 笔记模块 Typora 化改造计划

本计划旨在将现有的笔记模块改造成具有类似 Typora 的 **"所见即所得" (WYSIWYG)** 体验、强大的文件管理和高级 Markdown 支持的编辑器。

## 目标 (Goals)

1.  **核心编辑体验**: 实现源代码模式切换、数学公式/图表支持、图片相对路径存储。
2.  **文件管理**: 增强侧边栏，支持右键菜单（重命名、移动、在访达中显示）、排序、拖拽。
3.  **全局搜索**: 实现侧边栏实时搜索过滤。
4.  **导出功能**: 支持导出为 PDF。

## 阶段一：编辑器核心增强 (Editor Core)

这个阶段专注于 `MilkdownEditor` 组件的改造，使其功能对标 Typora。

### 1.1 源代码模式 (Source Code Mode)
Typora 的标志性功能之一是可以在“实时预览”和“源代码”之间无缝切换。
- **任务**:
    - 引入 `CodeMirror` 或简单文本框作为源代码编辑器。
    - 在工具栏添加 "源代码模式" 切换按钮。
    - 实现两种模式下的内容双向同步。

### 1.2 高级 Markdown 插件集成
- **任务**:
    - **数学公式**: 集成 `@milkdown/plugin-math` (KaTeX)。
    - **图表**: 集成 `@milkdown/plugin-diagram` (Mermaid)。
    - **表格**: 优化表格编辑体验，支持快捷键。
    - **GFM**: 确保 GitHub Flavored Markdown (任务列表、删除线等) 完美支持。

### 1.3 图片管理优化
Typora 能够将图片保存到相对路径 (如 `./assets`)，方便迁移。
- **任务**:
    - 拦截编辑器的 `Paste` 和 `Drop` 事件。
    - **文件系统模式**: 将图片自动保存到当前笔记同级的 `assets` 文件夹，并插入相对路径引用。
    - **数据库模式**: 继续使用全局图片存储或 Base64 (视现有逻辑而定，建议统一为应用数据目录存储)。

## 阶段二：文件管理与侧边栏 (File Management)

这个阶段专注于 `Notes` 页面侧边栏的重构，使其更像一个文件资源管理器。

### 2.1 增强型文件树
- **任务**:
    - 重构侧边栏文件列表，支持递归目录显示（如果尚未完美支持）。
    - **右键菜单 (Context Menu)**:
        - 重命名 (Rename)
        - 删除 (Delete) - 移动到回收站
        - 在文件资源管理器中显示 (Reveal in Finder/Explorer)
        - 复制文件路径
    - **排序功能**: 添加按“名称”、“修改时间”、“创建时间”排序的选项。

### 2.2 搜索与过滤
- **任务**:
    - 修复侧边栏搜索框。
    - **数据库模式**: 连接后端 `searchNotes` 接口，实时过滤列表。
    - **文件系统模式**: 实现内存中文件名过滤，若需全文搜索则需遍历文件 (暂定仅文件名过滤以保证性能，或利用 `ripgrep` 工具)。

## 阶段三：导出与 UI 优化 (Export & Polish)

### 3.1 导出功能
- **任务**:
    - 利用 Electron 的 `printToPDF` 或 `pandoc` (需用户安装) 实现导出。
    - 优先实现 **导出为 PDF** 功能 (最常用)。

### 3.2 UI 细节打磨
- **任务**:
    - 状态栏：显示字数统计 (Word Count)、阅读时间。
    - 优化工具栏布局，使其在“专注模式”下自动隐藏或更加极简。

## 执行路线图 (Roadmap)

1.  **Step 1**: 配置 Milkdown 插件 (Math, Diagram) 并验证。
2.  **Step 2**: 实现“源代码模式”切换逻辑。
3.  **Step 3**: 改造图片上传/粘贴逻辑 (核心痛点)。
4.  **Step 4**: 重构侧边栏，添加右键菜单和排序。
5.  **Step 5**: 完成搜索和导出功能。

---
*待确认事项*: 是否优先改进“文件系统模式”还是“数据库模式”？本计划默认两者并行推进，但侧重于文件系统模式的体验，因为这更符合 Typora 用户的使用习惯。
